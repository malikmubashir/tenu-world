# MOBILE-RUNBOOK — Tenu iOS + Android

Non-production scaffold delivered overnight 2026-04-17 → 2026-04-18.
This runbook is the exact command sequence to get the app onto a
simulator / device on your Mac. Read top to bottom.

---

## Prerequisites (one-time)

You need these on the Mac. Install if missing, skip if present.

```bash
# Xcode — from the Mac App Store. 15.x or 16.x.
xcode-select --install
sudo xcodebuild -license accept

# CocoaPods — iOS dependency manager.
sudo gem install cocoapods

# Android Studio — https://developer.android.com/studio
# Once installed, open it once → SDK Manager → install:
#   - Android 14 (API 34) SDK Platform
#   - Android SDK Build-Tools 34.x
#   - Android SDK Platform-Tools
#   - Android Emulator
#   - Google Play system images for API 34

# JDK 17 — required by Android Gradle Plugin 8.x
brew install --cask temurin@17

# Node 20 — the Vercel build already uses 20.x
node --version     # should print v20.x
```

Add to your `~/.zshrc` if not already there:

```bash
export JAVA_HOME=$(/usr/libexec/java_home -v 17)
export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator
```

## Step 2 — Build the mobile bundle

```bash
npm run build:mobile
```

This runs `MOBILE_BUILD=1 next build`, which:
- Enables `output: "export"` → writes static HTML to `./out`
- Disables the Next image optimizer (no sharp on device)
- Skips API routes (mobile calls `tenu.world/api/*` over HTTPS)

Expected output: `./out/` directory with all the `(mobile)/app-home/*`
pages statically rendered. If you see an error about `cookies()` or
`headers()`, a page under `(mobile)/` is trying to use server APIs —
grep for the call and wrap the data fetch in a client `useEffect`.

## Step 3 — Add the iOS platform

First time only:

```bash
npx cap add ios
```

Capacitor creates `./ios/App/` with an Xcode workspace. Then:

```bash
cd ios/App
pod install
cd ../..
```

Then apply the Info.plist additions:

```bash
open mobile/ios-Info.plist.snippet.xml
open ios/App/App/Info.plist
```

Copy each `<key>` / `<string>` pair from the snippet into the main
`<dict>` of Info.plist. Do NOT replace the whole file — Capacitor
already put mandatory keys in there.

Then in Xcode:

1. Open the workspace: `npx cap open ios`
2. Select the App target → **Signing & Capabilities**
3. Set Team to your Apple Developer team
4. **+ Capability** → **Associated Domains**
5. Add entries:
   - `applinks:tenu.world`
   - `applinks:www.tenu.world`
   - `applinks:app.tenu.world`
6. Replace `REPLACE_WITH_TEAM_ID` in
   `public/.well-known/apple-app-site-association` with your 10-char
   Team ID (shown in Xcode Signing pane), deploy to Vercel, and verify
   the file serves at `https://tenu.world/.well-known/apple-app-site-association`
   with `Content-Type: application/json` (headers added in next.config.ts).

Run:

```bash
npm run cap:ios
```

Xcode opens → pick iPhone 15 simulator → Cmd+R.

## Step 4 — Add the Android platform

First time only:

```bash
npx cap add android
```

Then apply the manifest additions:

```bash
open mobile/android-AndroidManifest.xml.snippet
open android/app/src/main/AndroidManifest.xml
```

Paste each element into the right parent tag per the comments in the
snippet. Also create `android/app/src/main/res/xml/network_security_config.xml`
with the content shown in the snippet, and add the
`android:networkSecurityConfig` attribute to the `<application>` tag.

Get the SHA-256 fingerprint of your signing certificate:

```bash
# Debug keystore (auto-generated by Android Studio on first run)
keytool -list -v -keystore ~/.android/debug.keystore \
  -alias androiddebugkey -storepass android -keypass android \
  | grep "SHA256:"

# Output: SHA256: AA:BB:CC:...  (64 hex chars with colons)
```

Paste that value (WITH the colons) into
`public/.well-known/assetlinks.json`, replacing
`REPLACE_WITH_SHA256_FROM_KEYSTORE`. Deploy to Vercel.

Run:

```bash
npm run cap:android
```

Android Studio opens. Pick Pixel 7 emulator API 34 → green play button.

For a production build you'll need a separate release keystore —
NOT the debug one. Post-launch work.

## Step 5 — What works today

- Local draft creation + photo capture (native camera on iOS/Android, HTML file picker on web preview)
- Photo sha256 hashing client-side
- Offline SQLite persistence of drafts + photos + upload queue
- Photo storage in Capacitor Filesystem (`Directory.Data`)
- Bottom tab nav (Inspections / Rapports / Réglages)
- Deep-link listener for `https://tenu.world/auth/callback*`
- `/api/mobile/upload-intent` and `/api/mobile/upload-commit` endpoints on the server

## What does NOT work yet (intentional scope cuts)

- **No end-to-end photo upload.** The sync engine is wired but stays
  idle because draft → server-inspection materialization is not
  implemented. The gap: `Review & Send` on
  `app-home/inspection/[id]/review/page.tsx` (page doesn't exist yet)
  must POST to `/api/inspection/create`, capture the returned
  `inspection_id`, call `attachServerInspectionId(draftId, id)` +
  `attachInspectionToDraftPhotos(draftId, id)`, then trigger the sync
  loop. Once those two helpers fire, photos drain automatically.
- **No auth on mobile.** Login / magic-link handoff deferred to
  Week 5. The scaffold's `Shell` registers the deep-link intent but
  no page consumes it yet.
- **No push notifications.** Capacitor Push plugin not installed.
- **No app icons or splash screens.** See task #31. Placeholder cream
  background only. Use `@capacitor/assets` to generate all sizes from
  a single 1024×1024 source PNG.
- **No localization.** UI is FR-only. `next-intl` not wired into the
  `(mobile)/` route group. Add `[locale]/` later once we decide whether
  to ship i18n day-one on mobile.

## Minimum env vars the device / server need

On Vercel (for the API endpoints):

```
R2_ACCOUNT_ID=<cloudflare account id>
R2_BUCKET_NAME=tenu-eu
R2_ACCESS_KEY_ID=<r2 key>
R2_SECRET_ACCESS_KEY=<r2 secret>
NEXT_PUBLIC_SUPABASE_URL=<supabase url>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<supabase anon>
SUPABASE_SERVICE_ROLE_KEY=<supabase service role>
```

On the mobile build (optional — defaults to tenu.world):

```
NEXT_PUBLIC_API_BASE_URL=https://tenu.world
```

For local iOS simulator development pointing at a local Next dev
server, temporarily set:

```bash
CAP_SERVER_URL=http://localhost:3000 npm run cap:sync
```

Then `npm run dev` separately, and the app loads from
`http://localhost:3000` instead of the bundled static export.
**Don't commit this env var** — it's only for local testing.

## Known gotchas

1. **`npm run build:mobile` fails with "Dynamic route requires
   generateStaticParams".** The `[id]` route needs an empty params
   array or `export const dynamicParams = true`. For the scaffold I
   rely on client-side routing only — if the build chokes, add
   `export const generateStaticParams = async () => []` to
   `inspection/[id]/page.tsx`.

2. **`pod install` fails with "Xcode command line tools".** Run
   `sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer`.

3. **Android emulator cannot reach `localhost`.** On Android emulator,
   the Mac is at `http://10.0.2.2:3000`, not localhost. Set
   `CAP_SERVER_URL=http://10.0.2.2:3000` for Android dev.

4. **Capacitor SQLite plugin fails on web with "not implemented".**
   Every DB call in `src/lib/mobile/storage/db.ts` returns null on
   non-native platforms. Callers guard with `if (db) { ... }`. If you
   add a new DB path, match that pattern.

5. **Universal Links don't fire first time.** iOS fetches
   `apple-app-site-association` on install and caches it. If you edit
   the file after install, delete the app and reinstall. Also verify
   `https://tenu.world/.well-known/apple-app-site-association`
   returns `200 OK` with `Content-Type: application/json` (no HTML
   wrapper — Vercel's auth wall on preview deployments would break
   this; only `main` deploys work for UL verification).
