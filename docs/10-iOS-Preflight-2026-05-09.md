## Stripe decision

`src/lib/mobile/checkout.ts:openStripeCheckout()` already uses `@capacitor/browser`
`Browser.open()` for native paths (`isNative() === true`) and falls back to
`window.location.href` for web. SFSafariViewController on iOS, Chrome Custom Tab on
Android. The `allowNavigation` whitelist in `capacitor.config.ts` does not need to
include `checkout.stripe.com` because Stripe Checkout is opened externally via the
system browser, not inside the WKWebView.

No code change required.

## AASA format note

The production route handler uses Apple's newer `appIDs` (plural) + `components` format
rather than the legacy `appID` (singular) + `paths` format. Both are valid. The newer
format supports glob-free path matching and is Apple's current recommendation. The
existing path set covers auth/callback, app-home, inspection, report, and dispute —
broader than the minimum required for magic-link auth, appropriate for post-launch
deep-link expansion.
