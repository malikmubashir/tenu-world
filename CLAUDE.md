## Task Tracker Protocol (added 2026-04-18)

Single source of truth for all Tenu work: **`TASKS.md`** at repo root. Every Claude surface (Cowork, Claude Code, Claude Chat, Claude Design) obeys this protocol.

### Files
- `TASKS.md` — master data source. Plain text, git-tracked, checkbox format, hand-edited or edited via Edit tool.
- `dashboard.html` — SSOT view (adopted 2026-05-21). Self-contained HTML, every open task rendered in full with `#NNN` ID, owner, priority, due date, full description. Auto-generated by `scripts/dashboard-build.py`. **Never hand-edit.** Dr Mubashir has it bookmarked. Refer to tasks by `#ID`.
- `dashboard-YYYY-MM-DD.html` — dated snapshot written alongside each rebuild for historical reference.
- `docs/Tenu-Reste-a-Faire.archive-2026-04-18.xlsx` — frozen pre-retirement snapshot with legacy Paperclip sheets (kept as audit archive, not maintained).

### Format inside TASKS.md
```
- [ ] OWNER: Title (p:N, due YYYY-MM-DD) — optional trailing note
```
States: `[ ]` open, `[>]` in progress, `[x]` done, `[-]` dropped.
Owners: `MH` (Dr Mubashir only), `CC` (Claude, any surface).
Priority: `p:0` blocks soft launch 11 May, `p:1` important, `p:2` post-launch.

### Start-of-session rules (every Claude surface)
1. Read `TASKS.md` first. No assumptions about state from prior sessions.
2. Identify what is relevant to the user's request. Confirm with user before starting if ambiguous.
3. Before touching code, mark the item `[>]` and add a dated note.

### Delegation rule (every Claude surface, added 2026-04-18)
Every action tied to work — whether Dr Mubashir delegates to Claude, Claude delegates back to Dr Mubashir, or either picks up a new item mid-conversation — **must** land as a line in `TASKS.md` before execution. No verbal-only tasks. No "I'll just quickly..." without a line.

**When I (Claude) delegate something to Dr Mubashir:** append an `MH:` line to the right section with `p:N`, `due YYYY-MM-DD`, and a crisp title. Tell him I added it.
**When Dr Mubashir hands me work:** append a `CC:` line, mark it `[>]` when I start, `[x]` with dated note when it ships.
**When a new task emerges from another task:** add it immediately, don't wait for end of session.

### End-of-session rules (Cowork + Claude Code)
1. Update task states in `TASKS.md`: `[x]` for done, `[>]` if in progress, `[-]` if dropped.
2. Add a `— done YYYY-MM-DD: <what shipped>` note on any closed item. The note must say what landed, not just "done".
3. Confirm every in-conversation task delegation has a corresponding TASKS.md line. If not, add it.
4. Run `python3 scripts/dashboard-build.py` to regenerate `dashboard.html` + the dated snapshot.
5. Commit together: `git add TASKS.md dashboard.html dashboard-$(date +%F).html && git commit -m "chore(tasks): ..."`.
6. Push to the active branch.

- At end, return a diff block the user can paste into Cowork or Claude Code. Do not pretend to have written the file.

### Never
